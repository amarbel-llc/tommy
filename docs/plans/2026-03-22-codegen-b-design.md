# Codegen B: Companion Document Type

## Problem

Tommy's reflection-based `pkg/marshal` cannot handle custom unmarshal types
(`Command`), pointer-to-struct fields (`*AnnotationFilter`), or pointer-to-
primitive fields (`*bool`). Downstream consumers like moxy must maintain adapter
structs that flatten these types into primitives, defeating the purpose of
type-safe round-trip TOML editing.

## Solution

A `//go:generate tommy generate` directive that produces a companion document
type pairing the user's struct with direct CST node references. The generated
code replaces runtime reflection with compile-time type-aware decode/encode
logic, handling all field types that the reflection path cannot.

## Architecture

    ┌─────────────────────────────────────┐
    │  cmd/tommy/generate.go              │  CLI subcommand entry point
    ├─────────────────────────────────────┤
    │  generate/                          │
    │    generate.go  — orchestrator      │  load package, find structs, emit files
    │    analyze.go   — struct analysis   │  go/ast + go/types → StructInfo tree
    │    emit.go      — field codegen     │  per-field decode/encode (fmt.Fprintf)
    │    template.go  — file skeleton     │  text/template for structure/signatures
    └─────────────────────────────────────┘

## Struct Discovery

The generator is invoked via `//go:generate tommy generate` in a Go source file.
It reads `$GOFILE`, loads the package with `golang.org/x/tools/go/packages`, and
finds all structs annotated with the directive comment
`//go:generate tommy generate` on the line immediately preceding the type
declaration.

Future option: process all exported structs with `tommy:` tags in the file
without requiring a per-struct directive.

## Analysis Model

The analyzer classifies each struct field into a `FieldKind`:

1.  **Custom** --- type implements `TOMLUnmarshaler` interface (checked first)
2.  **PointerStruct** --- pointer to a struct with `toml:` tags
3.  **Struct** --- nested struct with `toml:` tags
4.  **SliceStruct** --- slice of structs (array-of-tables)
5.  **SlicePrimitive** --- `[]int`, `[]string`
6.  **Primitive** --- `string`, `int`, `int64`, `float64`, `bool`
7.  **PointerPrimitive** --- `*bool`, `*int`, etc.

If a field type doesn't match any category, the generator emits a hard error at
generation time.

``` go
type StructInfo struct {
    Name   string
    Fields []FieldInfo
}

type FieldInfo struct {
    GoName    string
    TomlKey   string
    Kind      FieldKind
    ElemType  string
    TypeName  string
    InnerInfo *StructInfo
}
```

## Generated Code Shape

For a struct like:

``` go
//go:generate tommy generate
type Config struct {
    Servers []ServerConfig `toml:"servers"`
}

type ServerConfig struct {
    Name        string            `toml:"name"`
    Command     Command           `toml:"command"`
    Annotations *AnnotationFilter `toml:"annotations"`
}
```

The generator produces `config_tommy.go`:

``` go
// Code generated by tommy; DO NOT EDIT.

package config

type serverConfigHandle struct {
    node *cst.Node
}

type ConfigDocument struct {
    data    Config
    cstDoc  *document.Document
    servers []serverConfigHandle
}

func DecodeConfig(input []byte) (*ConfigDocument, error) { ... }
func (d *ConfigDocument) Data() *Config { return &d.data }
func (d *ConfigDocument) Encode() ([]byte, error) { ... }
```

### Key behaviors

- **Primitive fields:** Direct `document.GetFromContainer[T]` /
  `doc.SetInContainer` calls.
- **Custom unmarshal:** Calls `field.UnmarshalTOML(raw)` during decode using a
  new `GetRawFromContainer` document API. Calls `field.MarshalTOML()` during
  encode.
- **Pointer-to-struct:** Decode creates the struct only if the TOML table
  exists. Encode skips the section if the pointer is nil.
- **Pointer-to-primitive:** Decode sets the pointer only if the key exists.
  Encode skips if nil.
- **Array-of-tables:** Each entry gets an unexported handle holding its CST
  node. The public document type holds a slice of handles.
- **Nested structs:** Resolved recursively. Inner structs get unexported
  handles; only the top-level annotated struct gets a public document type.

### Companion type naming

- Public document: `<TypeName>Document` (e.g. `ConfigDocument`)
- Unexported handle: `<typeName>Handle` (e.g. `serverConfigHandle`)
- Decode function: `Decode<TypeName>` (e.g. `DecodeConfig`)

### Output file

Generated file is `<source_filename>_tommy.go` in the same package, with a
`// Code generated by tommy; DO NOT EDIT.` header.

## New Document APIs Required

Two new APIs on `document.Document`:

- `GetRawFromContainer(doc, node, key)` --- returns the TOML value as `any`
  without type conversion, for custom `TOMLUnmarshaler` types.
- `FindTableInContainer(doc, node, key)` --- finds a sub-table node within a
  container, for pointer-to-struct fields.

## Custom Type Interfaces

``` go
type TOMLUnmarshaler interface {
    UnmarshalTOML(data any) error
}

type TOMLMarshaler interface {
    MarshalTOML() (any, error)
}
```

The generator checks field types against these interfaces at generation time
using `go/types`. If a field type is not a recognized primitive, struct, slice,
or pointer variant and does not implement `TOMLUnmarshaler`, the generator fails
with a hard error.

## Code Emission Strategy

Hybrid approach:

- **`text/template`** for the file skeleton: generated header, imports, type
  declarations, function signatures.
- **`fmt.Fprintf`** for per-field decode/encode logic inside function bodies,
  where branching on `FieldKind` is complex.

## Comment Preservation

All comments in the original TOML survive round-trip because the generated code
holds direct CST node references and only mutates value nodes during `Encode()`.

New entries appended during encode (e.g. a new array-of-tables entry) will not
have comments --- there is no original CST to preserve. Programmatic comment
creation is deferred to Codegen C (issue #3).

## Rollback

Codegen B is purely additive:

- The reflection-based `pkg/marshal` remains fully functional.
- Consumers adopt generated companion types per-struct at their own pace.
- Rollback: delete `_tommy.go` files and remove `//go:generate` directives.
- No dual-architecture period or sunset timeline for `pkg/marshal`.

## Deferred

- Automatic discovery via `tommy:` tags (future alternative to per-struct
  directive)
- Programmatic comment creation (Codegen C, issue #3)
- Custom encode-only types (MarshalTOML without UnmarshalTOML)
